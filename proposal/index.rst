提案
=====
開放提案(Openness Proposal) 是一份支持任何語言和多種編譯語言的文件，最終通過統一的格式上傳至社區中。
提案是任何成員及世界上所存在的國家和地區的網民所提供的想法闡述渠道，任何具有可行性和可參與性的提案將會被理事會進行採納並最終實現，並根據不同的編號賦予其不同的功能。

提案編號
-----------
提案編號是在提案類型的前提下的延伸，聯合現在提案編號通常由類型簡寫加之時間中的年（取最後兩位數）、月組成作為年代索引並加上次號而定的，最後拼合為：``[type][year[day]][superposition]``。

子編號
-------
提案編號通常分為一級編號和二級編號，沒有在編號中融入二級編號索引的默認為一級編號。二級編號通常作為一級編號的擴展，針對單獨的問題所劃分的編號，以幫助區分信息。

子編號是需要根據提案來進行申請，並最終由去中心化工作流中對文件編號系統(JNUN File numbering system) 收入的才可成為子編號進行使用。每個子編號佔位一個格子。

即“``[alphabet]``” 一個編號中可以無限添加子編號以標識文件類型，最後拼合為：``[type][year[day]][superposition].[alphabet]···``。
